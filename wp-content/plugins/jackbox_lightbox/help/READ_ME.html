Special Notes

  • Due to local security restrictions, some things will not work locally, especially in Chrome.  But everything will work once the files have been updated to your website. I recommend testing in Firefox as is more lenient toward local iframe security. But the social buttons and gaussian blur hovers will only work online for all browsers.
  • Inside the download zip, you'll find two main folders. The first folder, named deploy is all you need to get started. This folder contains a single packed/minified version of all scripts.  The css files in the deploy folder are also minified.  If you wish to modify any of the scripts or CSS files, the original, unpacked/unminified versions can be found in the source folder.
  • JackBox uses deep-linking (every item has its own unique url) by default.  If your website already uses a deep-linking system, such as an Ajax-driven website, this feature will need to be turned off.  Refer to Global Settings to toggle this feature.
  • Autoplay for video and audio is not available on mobile due to device restrictions.
  • Mobile touch swipe functionality is only available for images
  • Fullscreen-mode is only supported in Chrome and Firefox.
  • In Chrome, when local HTML5 video is played, lightbox fullscreen mode is not supported. However, the HTML5 video player will have its own fullscreen button in Chrome. If having both functionalities is important, use the Flash version of the video player which has a mobile fallback.

Global Settings

JackBox only needs one line of code to get started:

jQuery(".jackbox[data-group]").jackBox("init");

But you can also override the default settings by passing a second argument into the jackBox() method.  This is demonstrated below where the individual settings are described:

jQuery(".jackbox[data-group]").jackBox("init", {

    deepLinking: true,              // choose to use the deep-linking feature ("true" will enhance social sharing!) true/false
    showInfoByDefault: false,       // show item info automatically when content loads, true/false
    preloadGraphics: true,          // preload the jackbox graphics for a faster jackbox
    fullscreenScalesContent: true,  // choose to always scale content up in fullscreen mode, true/false
    defaultShareImage: "1.jpg"          // if the social widget can't find an image to share from your lightbox content,
                                    // it will use this image's url instead.  Only applicable to Pinterest.

    autoPlayVideo: false,           // video autoplay default, this can also be set per video in the data-attributes, true/false
    flashVideoFirst: false,         // choose which technology has first priority for video, HTML5 or Flash, true/false
    defaultVideoWidth: 960,         // the default video width, can also be set per video in the data-attributes
    defaultVideoHeight: 540,        // the default video width, can also be set per video in the data-attributes
    
    useThumbs: true,                // choose to use thumbnails, true/false
    thumbsStartHidden: false,       // choose to initially hide the thumbnail strip, true/false
    thumbnailWidth: 75,             // the default video width, can also be set per video in the data-attributes
    thumbnailHeight: 50,            // the default video height, can also be set per video in the data-attributes
	useThumbTooltips: true,         // choose to use small thumbnail tooltips, true/false
    showPageScrollbar: false,       // if you don't like the "page jump" when the scrollbar disappears, set this to "true"
	useKeyboardControls: true			 // The left and right keyboard arrows will cycle through the items.  up/down will toggle thumbs
    
});

 

Item Settings

data-group              = The group/category, this is mandatory even for a single item
data-thumbnail          = Optional thumbnail to be used, original thumbnail will be used if this attribute is ommited.  
                          If tag is ommited and no original thumbnail exists either, if the item's JackBox content is an image,
                          the large version of the image will be used and scaled.
                          If none of these options are available, the "default thumbnail" jpg will be used.
data-title              = Optional item title, accepts text of html
data-thumbTooltip       = Optional attribute if you want to specify a thumbnail tooltip.  Otherwise the data-title will be used if available.
data-description        = Optional description, enter the id of the element where JackBox should grab the html/text from
data-scaleUp            = JackBox will always scale images down if necessary. Setting this attribute to true will scale the content up as well
data-width              = Override the default width (only applicable when content is not an image)
data-height             = Override the default height (only applicable when content is not an image)
data-flashHasPriority   = Applies to local video player, true = Flash w/ mobile backup, false = HTML5 with Flash backup
data-firefoxUsesFlash   = Set this to true if you wish to use the HTML player w/ Flash fallback but want Firefox to use Flash no matter what
                          This is useful when you want to use the HTML5 version of the player but don't want to create a webm fallback for Firefox
data-poster             = Applies to local video, will be shown for mobile devices that don't support autoplay (when we can't grab the first frame of the video)
data-autoplay           = Applies to video and audio
data-audiotitle         = The audio player song title
href/data-href          = the content to be loaded inside JackBox

 

JackBox Examples

Custom HTML / Single Item Example

Although JackBox supports grouping/categories, you may just want to have a single item appear in the lightbox.  Here is an example:

<a class="jackbox" data-group="info" data-title="JackBox Just Works" data-width="640" href="#learn-more">Learn More</a>

The above example is just a text link that loads an html div.  Even though we're just loading a single item, a data-group is still required.  This allows for the creation keyword enhanced urls.
For the above example, the following would be the url based on the data-group attribute:

www.yoursite.com#/info/

The data-title is optional.  Below is a screenshot of the data-title

data-width is optional.  But because the default width for video, iframes and html content is 960, we can override that number by including the data-width attribute.

And finally, the href tag links to the id of the html div we wish to display for this example.  Because this JackBox item is a link in nature, we'll use the href attribute.  but a JackBox item doesn't need to be a link.  It can be any tag, and in those cases, instead of using the href attribute, a data-href attribute would be used instead.

Image Example

The following is a simple example of loading an image inside JackBox. 

<a class="jackbox"

    data-group="images"
    data-thumbnail="jackbox/img/thumbs/1a.jpg"
    data-title="Image Title with <a href='http://www.google.com' target=_blank'>link</a>"
    data-description="#description_1"
    href="jackbox/img/photos/1.jpg"
    
><!-- end opening tag -->

    <!-- original thumbnail -->
    <img src="jackbox/img/thumbs/1.jpg" width="180" height="150" alt="responsive lightbox" />
    
</a>

The data-thumbnail attribute is an optional attribute that can be added to force JackBox to use a specific thumbnail url when JackBox is in "gallery mode" (i.e. when there are multiple items inside a data-group).  But a seperate, custom thumbnail is NOT required.  Instead, if the element on your page that triggers the lightbox is a thumbnail to begin with, JackBox can simply use that thumbnail instead (screenshot below)

Would then be resized and used here:

But what if the JackBox item is a simple link, like shown in the Custom HTML section?  In these cases, JackBox will then see if the lightbox content to be displayed is an image.  If so, it will then resize and use that image as the thumbnail.  And finally, if a large image is not available, JackBox will then use the default thumbnail which is located inside the jackbox/img/thumbs folder and is named default.jpg.

The data-description attribute is the id of the html div you wish to use as your item's description.  This div can be placed anywhere inside your web page, as it will be hidden from view automatically. 

<div class="jackbox-description" id="description_1">

    <h3>Description Title</h3>

</div>

The contents of the div will them be used as the item's description (screenshot below)

To autoplay videos, use the special data-autoplay attribute.  Or you can set autoplay as the default behaviour inside the Global Settings.

Local Video

There are a few custom modules that JackBox uses.  Local video is one of them.  Here is a local video taken directly from one of the included html files:

<a class="jackbox"

    data-group="video"
    data-thumbnail="jackbox/img/thumbs/7a.jpg"
    data-title="HTML5 Video w/ Flash Backup"
    data-description="#description_1"
    data-flashHasPriority="false"
    data-autoplay="true"
    data-width="958"
    data-height="538"
    data-poster="jackbox/img/video_poster.jpg"
    href="jackbox/video/1.mp4"
    
><!-- end opening tag -->

    <img src="jackbox/img/thumbs/7.jpg" width="180" height="150" alt="responsive lightbox" />
    
</a>

data-flashHasPriority is one of the cool features about JackBox local video.  The video player doubles as an HTML5 player AND a Flash Player.  Both are needed to make sure your video can be accessed from all devices/browsers.  For example, IE8 doesn't support HTML5 so we'll always use the Flash version for that.  And iPad/iPhone's don't support Flash, so the HTML5 version will always be used for those devices. 

But for browsers that support both, such as Firefox and Chrome, you have the ability to choose which version of the player should have priority in these cases.  If you'd like HTML5 to have priority, set this attribute to false.  Or if you'd like Flash to have priority, set this attribute to true.

data-autoplay is an optional attribute that will allow you to turn on autoplay per video.  If all of you videos will either autoplay or not autoplay, use the global autoplay setting instead (see Global Settings).

data-poster is a special attribute only used for local video.  The HTML5 player will always just grab the first frame of the video and use that as the poster.  But because iPhones/iPads don't support autoplay, we can't grab the first frame of the video.  And in these cases, the data-poster will be displayed instead. 

Why is the first frame of the video used instead of the data-poster when autoplay is supported?  Because starting and stopping the video on the first frame allows us to begin the loading process.  This allows us to begin preloading the video immediately leading to less interrupted playback.  But because the video has technically played, HTML5 technology will subsequently hide the poster by default.

data-width and data-height are important attributes for the Audio module.  This is because the Audio module has its own background image.  To change this, open the file named jackbox/css/jackbox-audio.css, and at the very top you'll see the following code where the 500x185 background image is declared.  If you choose to change the data-width and data-height, be sure to change this background image as well.

body {

    background: #000 url(../img/audio_poster.jpg);
    
}

The Audio module also uses Google Fonts.  Refer to Changing the Google Font for an example of how to change this.  If you choose to change this font, the files that need to be edited are jackbox/modules/jackbox_audio.html and jackbox/css/jackbox-audio.css.

data-audiotitle is a special attribute used only by the Audio module.  This attribute accepts plain text or hyperlinks.  The example above hyprlinks the text.

Just like the video examples, the Audio module also supports the autoplay attribute.

Hidden Items

Similar to other lightboxes, JackBox allows you to create groups/categories of items.  This can be done two ways:

  1. Use the same data-group for all the items on your page.  This is usually used for gallery-type pages where all images belong to the same category.
  2. Create a list of hidden items that will only be shown when a particular item is selected.  This is useful for portfolio-type pages where you'd like to show additional images of a project.

To create a list of hidden items, simply create and paste an unordered list below your JackBox item.  Here's an example:

<a class="jackbox" data-group="vimeo" data-title="Surrealist Dollhouse" data-description="#description_1" href="http://vimeo.com/17573879">
    
    <img src="jackbox/img/thumbs/8.jpg" width="180" height="150" alt="responsive lightbox" />

</a>
 
<ul class="jackbox-hidden-items">
    
    <li class="jackbox" data-group="vimeo" data-title="Kilogramme" data-description="#description_1" data-href="http://vimeo.com/17516687"></li>
    <li class="jackbox" data-group="vimeo" data-title="Dreamland Sender" data-description="#description_1" data-href="http://vimeo.com/17586987"></li>
    <li class="jackbox" data-group="vimeo" data-title="Google eBooks" data-description="#description_1" data-href="http://vimeo.com/17574435"></li>
    <li class="jackbox" data-group="vimeo" data-title="Henning Sandström" data-description="#description_1" data-href="http://vimeo.com/35416846"></li>
    <li class="jackbox" data-group="vimeo" data-title="Nightscapes" data-description="#description_1" data-href="http://vimeo.com/35406208"></li>
    <li class="jackbox" data-group="vimeo" data-title="Nothing To Fear" data-description="#description_1" data-href="http://vimeo.com/17593192"></li>
    <li class="jackbox" data-group="vimeo" data-title="Refraction" data-description="#description_1" data-href="http://vimeo.com/16980912"></li>

In the example above, the first part (the hyperlink) represents the thumbnail that will be visibly present on your web page.  This is what the user would click to activate the lightbox. 

The second part represents the hidden items that will ONLY appear inside the lightbox once activated.  Technically this unordered list can exist anywhere on your web page, as the unordered list itself will be hidden by default. 

Because a JackBox item can be any tag, we can organize the hidden items as list items.  As you can see, all of the data-attributes are the same, with the only exception being that the list items use the data-href attribute instead of the href attribute. 

All of the items in the example above are automatically grouped together in the lightbox by having the same data-group.

Retina Graphics

Some of the graphics used in JackBox are CSS based, and in these cases, they're "retina-ready" by default.  But most of the buttons/icons used in the lightbox are png images. 

Here's a good article that talks about "retina" and how it applies to us webmasters:

http://coding.smashingmagazine.com/2012/08/20/towards-retina-web/

The summary is that all "retina-ready" graphics/icons need to be double in size.  In the download source, open the jackbox/img/retina folder for an example.  With this special folder, all icons have two versions:  one double the size, and one regular size.  And CSS media queries are used to determinne which version of the icon should be displayed. 

Open the file jackbox/css/jackbox.css and scroll to the bottom.  Here you'll find the following media query:

@media
only screen and (-webkit-min-device-pixel-ratio: 2),
only screen and (min--moz-device-pixel-ratio: 2),
only screen and (-o-min-device-pixel-ratio: 2/1),
only screen and (min-device-pixel-ratio: 2),
only screen and (min-resolution: 192dpi),
only screen and (min-resolution: 2dppx) {

Everything inside that query is what will be used for screens that are "retina-display".  You'll also find retina media queries at the bottom of the files jackbox/css/jackbox-video.css and jackbox/css/jackbox_hovers.css files. 

The audio module is 100% CSS and is retina-ready by default.

Responsive CSS

Inside the CSS files toward the bottom you'll find media queries for smartphones.  Here's an example:

@media screen and (max-width: 600px) {
    
    .jackbox-panel,
    .jackbox-thumb-holder {
       
        display: none !important;
       
    }
    
    .jackbox-show-thumbs,
    .jackbox-hide-thumbs {
    
        width: 0;
        margin-left: 0;
        overflow: hidden;
       
    }
    
}

This example pertains to a smartphone-landscape screen.  And in this example, the large prev/next buttons that are normally shown on the left and right side of the screen are hidden as well as the lightbox thumbnails.  This leaves the necessary room needed for the actual content to display properly on small screens.

Working with HTML5 Video and Audio

JackBox supports HTML5 Audio and Video.  The default file type for video is mp4 and the default for audio is mp3. 

Firefox, in particular, does not support mp3 or mp4.  Because of this, alternative file-types need to be created.  For video, JackBox is designed to fall back to the webm file-type when mp4 is not supported. For audio, JackBox will fall back to the ogg file-type when mp3 is not supported. 

Because of this, every audio and video file must have two versions:

  • The default ("mp3" for audio, "mp4" for video)
  • The fallback ("ogg" for audio, "webm" for video)

If you look inside the music and video folders, you'll see that each item has both a default and a fallback version.  For the fallback to work properly, the media item's name must be identical for each version.  Only the file's extention should be different.  Here's an example:

  • song.mp3
  • song.ogg

When editing the html files, always use the default file-types (mp3, mp4) for the media urls. JackBox will then dynamically replace the mp3 or mp4 versions with the ogg or webm fallbacks when neeeded. 

To create the ogg and webm versions of your media, I recommend the following tools:

  • http://media.io/ (mp3 > ogg conversion)
  • http://www.mirovideoconverter.com/ (mp4 > webm conversion)

The traditional "flv" video file type's are not needed even for the Flash fallback.  This is because the Flash fallback can play the "mp4" file-type.

You may find that the audio and video works locally in Firefox but not online.  If you experience this, it likely means that your server has not updated to support the latest HTML5 media extentions.  But you can manually update your server.  Here's how:

  1. Download the .htaccess file from the root directory of your server.  If one does not exist, create a new text file on your computer and name it htaccess.txt.
  2. If you were able to find the .htaccess file on your server, rename it to "htaccess.txt".  Your computer may alert you about changing the file's extention.  Just click "ok" if you get this message.
  3. Next, open htaccess.txt in your favorite text editor, and add the following to the bottom of the file:

    AddType audio/ogg .ogg
    AddType video/webm .webm
     
  4. Next, save the file, and upload it to the root directory of your server.  Then delete the previous .htaccess file from your server, and rename the htaccess.txt file you just uploaded to .htaccess.

Under the Hood / API Methods

Most things can be changed by the methods discussed throughout this help document.  But some settings are declared inside JackBox's core script.  To view them, open the file jackbox/js/jackbox.js inside the source folder, and at the very top you'll some of these settings along with comments that describe each one.

API Methods

JackBox has an API method that makes it easy to add an item to the lightbox list at any time.  Here's a basic example with only required params:

// Always use the the jackBox.available() wrapper to make sure JackBox is available
jQuery.jackBox.available(function() {
    
    jQuery("<div />").html("New JackBox Item").css("cursor", "pointer").appendTo(jQuery("body")).attr({
       
        "data-group": "info",
        "data-href": "jackbox/img/video_poster.jpg"
       
    }).jackBox("newItem");
    
});

And here's an example with all available options shown:

// Always use the the jackBox.available() wrapper to make sure JackBox is available
jQuery.jackBox.available(function() {
    
    // just an example new item to be added
    var div = jQuery("<div />").html("New JackBox Item").css("cursor", "pointer").appendTo(jQuery("body"));
    
    // If the element doesn't have the appropriate data-attributes already applied,
    // they can be explicitly set in the settings parameter as shown below
    
    // If you want to run the JackBox item right away, add a "trigger" param and set it to true,
    // Otherwise, the element will be added to JackBox and can be triggered later by a user click
    
    div.jackBox("newItem", {
       
        group: "info",
        title: "JackBox Just Works",
        href: "jackbox/img/video_poster.jpg",
        trigger: true
       
    });
    
});

To trigger a JackBox item manually once it's been added to the lightbox, you can use the following jQuery technique:

jQuery.jackBox.available(function() {
    
    jQuery("#someItem").trigger("click");
    
});

JackBoxFX

JackBoxFX are one of the bonuses included with JackBox.  There are 8 different effects that you can create. 

Some notes:

  • The JackBoxFX plugin is completely independent of JackBox (but can also be used with JackBox).
  • The thumbnail's parent element will automatically adopt a position of relative in order for the effect to work properly.
  • The thumbnail's parent element must also have a display other than inline.  This means it should either be block or inline-block.
  • If the thumbnail's parent element has display set to inline, inline-block will be automatically applied
  • The FX call is for IMAGES only. In the example below, any image with the class name jackbox-fx will get the chosen effect.
  • Not all effects are supported cross-browser. View the jackbox_fx_all.html in your web browser file for more information.

To add an effect, first add a custom class of id to the thumbnail itself.  Here's an example:

<img class="jackbox-fx" src="jackbox/img/graphics/pixel.png" data-src="jackbox/img/thumbs/6.jpg" width="180" height="150" alt="JackBoxFX" />

Notice that the above example has a fake src, and the real url to the image is placed in the data-src attribute.  This is because we need to load the image before the effect can be created with HTML5.  JackBoxFX will take care of this automatically.

The one exception to the data-src rule above is the zoom effect.  This is what a zoom effect looks like:

<img class="jackbox-fx" src="jackbox/img/thumbs/1.jpg" data-zoom="jackbox/img/zooms/1.jpg" width="180" height="150" alt="JackBoxFX" />

In the zoom example above, the src is used for the normal thumbnail url, and a data-zoom atrribute is used for the larger version of the thumbnail.  The data-zoom attribute needs to be an image that is exactly 2x the size of the original.

Here are the different JackBoxFX available:

  1. zoom
  2. shine
  3. blinds
  4. rubix
  5. scramble
  6. rotate
  7. flip
  8. scale

Once the image tag has been properly created, attaching an effect is easy.  Here's a simple example.

$(document).ready(function() {
    
    $(".jackbox-fx").jackboxFX("zoom");
    
});

The first part is the class we added to the image tag itself.  The second part is the FX method.  This method accepts two arguments.  The first is the effect type which is mandatory.  The second argument is for the optional settings.  Here's an example:

$(document).ready(function() {
    
    $(".jackbox-zoom").jackboxFX("zoom", {
        
        // applies to the shine effect
        shineContrast: 1.5,
        shineBrightness: 30,
        
        // applies to the zoom effect
        zoomBorder: "1px solid rgba(0, 0, 0, 0.7)",
        zoomBorderIE: "1px solid #000",
        
        // applies to the rubix effect
        rubixBorder: "3px solid rgba(0, 0, 0, 0.75)",
        rubixBorderIE: "3px solid #000",
       
        cols: 4,// only applicable to effects numbered 3-8 above
        rows: 3, // only applicable to effects numbered 3-8 above
        color: "#FFF", // only applicable to the "blinds" and "rubix" effect
        reversed: false // only applicable to the "scramble" effect
       
    });
    
});

 

Standalone Video Player

Before reading this section, first open the following files:

  • standalone_html5_video_player.html
  • standalone_flash_video_player.html

These files are virtually identical, except one has flashing (Flash Priority) set to true and the other has flashing (Flash Priority) set to false.  Check out Local Video under the JackBox Examples for a more detailed description about Flash Priority.

The video player is loaded as an iFrame, similar to how YouTube and Vimeo works.  Here's an example:

<iframe
      
  type="text/html"
  width="100%"
  height="100%"  
  frameborder="0"
  scrolling="no"
  mozallowfullscreen="mozallowfullscreen"
  webkitallowfullscreen="webkitallowfullscreen"
  allowfullscreen="allowfullscreen"
  src="jackbox/modules/jackbox_video.html?video=video/3.mp4&autoplay=false&flashing=true&width=960&height=540&poster=img/video_poster.jpg"
      
></iframe>

The src part is the one that will need to be edited.  The first part of the src links to the video module itself:

jackbox/modules/jackbox_video.html

The following are the url parameters that need to be set:

video=video/3.mp4 // the url to the mp4 version of the video

autoplay=false // choose autoplay, true/false

flashing=false // choose whether Flash has priority (see notes above)

width=960 // the video's original width

height=540 // the video's original height

poster=img/video_poster.jpg // the fallback poster when autoplay isn't available

All of the url parameters are mandatory and their order should not be changed
